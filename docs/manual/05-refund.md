# 05. 거래 상세 · 환불

**경로:** 분할정산 거래내역(`/settlement/transactions`) → 행의 **환불** 버튼 → 환불하기 모달

## 화면 개요

거래 상세 내역에서 **환불**을 실행하는 기능입니다. 가맹점 거래(TID)와 셀러 거래(STID)의
환불 범위가 다릅니다.

> - **TID 기준**: 전체 환불만 가능(`전체 환불 only`). 환불 클릭 시 **전체환불 / 부분환불 분기**.
> - **STID 기준**: **부분 / 전액 환불** 가능.

## 환불하기 모달 구성

### 1. 거래 기본정보 영역
- 거래번호(TID)
- **총 거래 금액**
- **환불 가능 금액(잔액)**

### 2. 환불 영역 — 전체환불 / 부분환불 (Radio)

#### 2-1. 전체 환불
- 총 환불 가능 금액을 **텍스트로 노출**
- **[총 금액 환불하기]** 버튼 → 클릭 시 총 금액 환불 진행
- 안내: *"전체 거래 금액이 환불됩니다. 조정이 필요하면 부분환불을 이용하세요."*

#### 2-2. 개별(부분) 환불
- **STID별 영역**: STID / 셀러명 / 거래금액 / 환불 가능 금액 노출 + **체크박스**
- 체크박스 클릭 시 → **환불 금액 입력 필드** 노출
  - 환불 가능 금액을 **초과 입력하면 환불 가능 금액으로 자동 보정**
  - **[전액]** 버튼 클릭 시 → 환불 가능 금액 총합이 입력창에 채워짐
- **[환불하기]** 버튼
  - 라벨: *체크한 STID의 입력 환불 금액 총합* + "환불하기" (예: `NN개 총 $환불SUM$ 환불하기`)
  - 클릭 시 → **확인 모달** 노출
  - 확인 모달에서 [환불하기] 클릭 시 → **STID + 환불 금액으로 환불 API 호출**

### 확인 모달
- *"선택한 NN개 환불을 실행합니다. 실행 후 취소할 수 없습니다."*

## 동작 요약 (플로우)

```
거래 상세 → [환불] 클릭
   ├─ 전체환불 선택 → [총 금액 환불하기] → 확인 모달 → 환불 API
   └─ 부분환불 선택 → STID 체크 → 금액 입력(초과 시 보정) → [환불하기]
                         → 확인 모달("실행 후 취소 불가") → 환불 API
```

## 예외 · 유의사항

- **STID 없는 TID는 환불 불가.**
  - 안내: *"환불 처리를 진행할 수 없습니다 / 셀러 거래내역이 들어와야 환불이 가능합니다."*
  - 즉, 셀러 분할 내역(STID)이 적재된 거래만 환불 대상이 된다.
- 환불은 **실행 후 취소 불가**(확인 모달에서 재확인).
- 부분환불 입력 금액은 **STID별 환불 가능 금액**을 초과할 수 없다(초과 시 자동 보정).

## 설계 메모 (기획서)

- STID 거래내역과 환불/로그 기능을 **별도 화면으로 분리**하는 방향(현행 ver).
  - 분리 시 장점: VOC 환불 동선([02 §VOC](./02-transactions.md))에서 동작이 단순.
  - 비분리 시 장점: TID 기준으로 STID 환불을 함께 볼 수 있음.
