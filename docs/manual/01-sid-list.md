# 01. SID 리스트

**경로:** 셀러 > 분할정산 > SID 리스트 · 데모 `/settlement/sids`

## 화면 개요

분할정산 대상 셀러(SID) 등록 현황을 조회·관리하는 화면입니다.
**송금 서비스와 분할정산을 통합**한 SID 목록이며, SID 등록은 세 가지 방식으로 들어옵니다.

> **SID 등록 방식:** `API` / `단건등록` / `일괄등록`

## 화면 구성

### 필터
- **날짜 범위** (최초접수일 기준)
- **진행 상태** 드롭다운 — 접수완료 / 심사중 / 승인 / 반려
- **검색어** — MID · sellerID · 법인명 · 대표자명
- **초기화** / **다운로드**

### 목록 컬럼
| 컬럼 | 설명 |
|------|------|
| NO | 행 번호 |
| 가맹점 ID (MID) | 셀러가 속한 가맹점 |
| sellerID | 셀러 식별자 |
| 법인명 | 셀러 법인명 |
| 대표자명 | 셀러 대표자 |
| 진행 상태 | 접수완료 / 심사중 / 승인 / 반려 |
| 최초접수일 | 등록 접수일 |
| 최종수정일 | 마지막 수정일 |
| **MID 생성** | 셀러용 MID 생성 액션 |

## 동작

1. 진입 시 SID 목록을 조회한다(최초접수일 내림차순).
2. 검색어·진행 상태·날짜 범위로 필터링하면 목록과 페이지네이션이 즉시 갱신된다.
3. **MID 생성**: 진행 상태가 **`승인`** 인 건만 활성화되며, 클릭 시 해당 셀러의 결제 MID를 생성한다.
   - 이미 생성된 셀러는 `생성완료`로 표시된다.
4. **다운로드**: 현재 조회 조건의 결과를 파일로 내려받는다.

## 예외 · 유의사항

- 진행 상태가 `승인`이 아니면 **MID 생성 불가**(버튼 비활성).
- 셀러를 영중소로 취급하려면 가맹점 정보 > 셀러 탭에서 **영중소(Y/N)** 값을 `Y`로 설정해야 한다
  (자세한 설정은 [03 정산 리스트](./03-settlement-list.md) · [04 영중소 정산서](./04-sme-settlements.md) 참고).
