# 01. SID 리스트

**경로:** 셀러 > 분할정산 > SID 리스트 · 데모 `/settlement/sids`

> 표기 규칙
> - ✅ **기획 근거**: Figma 화면/주석 텍스트에 실제로 존재하는 내용
> - ⚠️ **데모 임의 구현**: 기획에 명시가 없어 데모에서 임의로 정한 부분(스펙 아님)

## 화면 개요

분할정산 대상 셀러(SID) 등록 현황을 조회하는 화면입니다.
✅ 메뉴 주석상 **SID 리스트는 "송금 서비스 & 분할 정산 통합"** 목록입니다.

✅ **SID 등록 방식** (주석): `API` / `단건등록` / `일괄등록` — "API 통신으로 일괄 등록 가능".

## 화면 구성

### 목록 컬럼 (✅ 화면의 컬럼 헤더)
NO · 가맹점 ID (MID) · sellerID · 법인명 · 대표자명 · 진행 상태 · 최초접수일 · 최종수정일 · MID 생성

- **MID 생성**: 화면에 셀러용 MID를 생성하는 액션(셀/버튼)이 있음.
- **진행 상태**: 컬럼은 존재하나, **상태 값 목록은 Figma 텍스트에 명시되어 있지 않음**.

### 상단/필터 (✅ 화면 스크린샷)
- **초기화**, **다운로드** 버튼
- 날짜 범위 + 드롭다운 필터 영역(와이어프레임의 필터 라벨은 템플릿 상태)

## 동작 (기획 근거 있는 것만)

- ✅ SID 등록은 API / 단건등록 / 일괄등록으로 들어온다.
- ✅ 목록에서 MID 생성 액션을 제공한다.

## ⚠️ 데모 임의 구현 (기획 주석 근거 없음)

아래는 **Figma 주석에 문장으로 존재하지 않으며**, 데모를 동작시키기 위해 임의로 정한 규칙입니다.
실제 스펙은 디자인의 버튼 컴포넌트 상태(variant) 또는 Confluence 원본 기획서에서 확인해야 합니다.

- 진행 상태 값을 `접수완료 / 심사중 / 승인 / 반려`로 정함 → **주석 근거 없음**
- "MID 생성은 진행 상태가 승인인 건만 활성화", "이미 생성된 건은 생성완료 표시" → **주석 근거 없음**
- 목록 정렬(최초접수일) → **주석 근거 없음**

## 참고 (✅ 다른 화면 주석)

- 셀러를 영중소로 취급하려면 가맹점 정보 > 셀러 탭에서 **영중소(Y/N)** 설정이 필요
  (주석: "셀러를 영중소로 취급 희망 > 영중소(Y/N)값을 Y로 설정 필요"). → [04 영중소 정산서](./04-sme-settlements.md)
- 원본 기획서 본문 일부는 Confluence(`erpayverse.atlassian.net`)에 있어 외부 접근이 제한됨.
