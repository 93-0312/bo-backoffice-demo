# 02. 분할정산 거래내역

**경로:** 셀러 > 분할정산 > 거래내역 · 데모 `/settlement/transactions`

## 화면 개요

**STID 단위**의 셀러 거래내역을 조회하는 화면입니다. 하나의 가맹점 결제(TID)가 여러 셀러(SID)에게
분할되면 각 셀러 몫이 STID로 기록됩니다. VOC(환불 문의) 대응의 출발점이 되는 화면입니다.

## 화면 구성

### 필터
- **날짜 범위** — **최대 검색가능 기간: 1개월**
- **거래 유형** — 승인(SALE) / 환불(REFUND)
- **거래 상태** — 성공(SUCCESS) / 거절(DECLINE) / 대기(PENDING)
- **검색어** — STID · SID · 셀러명 · 가맹점명 · TID
- **페이버스 결제 제외** 토글 — 페이버스 자체 결제 건 제외
- **초기화** / **다운로드**

### 목록 컬럼
NO · STID · 거래일시 · 셀러주문ID · SID · 셀러명 · 가맹점ID · 가맹점명 · TID · 주문ID · 거래 유형 · 거래 상태 · 통화 · 금액

## 동작

1. 기간(최대 1개월)·유형·상태로 거래를 조회한다.
2. **페이버스 결제 제외**를 켜면 페이버스 결제 건이 목록에서 빠진다.
3. 행의 STID/거래상태로 환불 필요 여부를 판단한다. **행을 클릭하면 거래 상세 내역**으로 이동한다.
4. 환불 처리는 목록이 아니라 **거래 상세 내역 화면의 환불 버튼**에서 진행한다 ([05 환불](./05-refund.md)).

### VOC 기반 환불 처리 동선 (실제 운영 플로우)

> 플랫폼에 셀러 A의 "환불이 안 됐어요" VOC가 접수된 경우:
> 1. 플랫폼 담당자가 **BO 접속**
> 2. **거래내역 목록에서 검색** → 셀러명 · STID · 주문ID 중 하나로 검색
> 3. 해당 **STID 건 확인** → 거래 상태 체크 (SUCCESS / DECLINE / REFUND)
> 4. 환불 처리가 필요하면 환불 액션 수행

## 예외 · 유의사항

- 검색 기간은 **최대 1개월**을 넘길 수 없다.
- 환불은 셀러 거래내역(STID)이 존재해야 가능하다 → STID가 없는 TID는 환불 불가
  ([05 환불 §예외](./05-refund.md) 참고).
