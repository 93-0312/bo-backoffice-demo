## 메뉴 구조 (현행 가능 구조)

```
셀러 정산관리
├── SID 리스트              (송금 서비스 & 분할정산 통합)
├── 송금 서비스 거래내역
└── 분할정산 거래내역        (= 셀러 거래내역)

가맹점 정보
└── 가맹점(기본) > 가맹점 상세
    └── 셀러 탭 신설         (기존 하위셀러 내용 포함)

정산서
├── 정산서(신규)            (셀러 일반 정산 포함)
└── 영중소 정산서 목록       (셀러 영중소 정산서 포함)
```

## 핵심 용어

| 용어 | 의미 |
|------|------|
| **MID** | 가맹점 ID. 결제 가맹점 식별자. |
| **SID** | 분할정산 대상 셀러 식별자 (MID 하위). 과거 명칭 SMID. |
| **STID** | 셀러 거래 단위 식별자. 하나의 가맹점 거래(TID)가 여러 STID로 분할됨. |
| **TID** | 가맹점 결제 거래 식별자. |
| **분할정산** | 한 결제를 여러 셀러(SID)에게 나눠 정산. |
| **차액정산** | 정산 과정에서 발생한 차액(수수료·환불 등)을 정산. |
| **영중소(SME)** | 영세·중소 가맹점. 우대 수수료/별도 정산서 대상. |
| **정산방식** | `MID정산`(가맹점 단위) / `셀러별정산`(SID 단위). **현행은 MID정산만 제공**, 셀러별정산은 차후. |

## 공통 규칙

- **검색 기간**: 거래내역 최대 **1개월**, 정산/영중소 정산서 최대 **12개월**.
- **페이버스 결제 제외**: 거래내역에서 페이버스 자체 결제 건을 제외하는 토글.
- **다운로드**: 각 리스트는 현재 조회 결과를 내려받을 수 있음(엑셀).
- **페이지네이션**: "N 내역 중 a ~ b 표시" + N개씩 보기.

## 근거 표기 규칙 (중요)

이 메뉴얼은 **Figma 화면/주석 텍스트에 실제 존재하는 내용만** 스펙으로 적습니다.

- ✅ **기획 근거**: Figma 화면(컬럼/버튼)·주석 텍스트에 존재
- ⚠️ **데모 임의 구현**: 기획에 명시가 없어 데모에서 임의로 정한 값/규칙(스펙 아님)

> **정정 이력:** 초기 작성 시 일부 동작 규칙(예: SID 리스트의 "진행 상태 = 접수완료/심사중/승인/반려",
> "MID 생성은 승인 건만 활성화/생성완료 표시", 정렬 순서, 거래 상태 `PENDING`, 정산 상태 `지급 대기`)을
> **Figma 주석에 없는데 추론으로 적었던 오류를 발견·정정**했습니다. 전체 텍스트 노드 402개를 재검색해 확인.
> 그런 항목은 각 문서에서 ⚠️ 로 명시했습니다.

> ⚠️ 원본 기획서 본문 일부는 Confluence(`erpayverse.atlassian.net`)에 있으며 외부 접근이 제한됩니다.
> 데모 구현값(상태 enum 등)의 정확한 스펙은 디자인 컴포넌트 variant 또는 Confluence 원본에서 확인이 필요합니다.
